# MICTF —— 形态信息驱动的接触模板框架

**在线、无需训练数据的重定向方法，在精确捏合建立*之后*维持 pad 级接触。**

> *Morphology-Informed Contact Templates for Online Contact-Maintaining Dexterous Retargeting*
> 📄 **状态：** 准备投递 IEEE RA-L。
> 💬 讨论与提问：[GitHub Discussions](../../discussions) · 问题反馈：[GitHub Issues](../../issues)

## 这项工作为什么存在

绝大多数灵巧手重定向方法 —— DexPilot 式向量跟踪、关节复制、指尖 IK —— 回答的都是同一个问题：**"指尖该去哪里？"** 它们在自由空间里表现得很好。可一旦两个 pad 已经接触、操作者还在继续移动，这些表征没有任何显式机制来**保证接触面持续贴合**。结果：指尖留在向量目标附近，但接触悄悄被撑开。

这发生的原因是结构性的。一个仅被 3D 指尖向量约束的 4-DOF 机器人拇指，存在一个未约束的自由度 —— pad 绕"拇指-指尖轴"的旋转。DexPilot 的平滑正则项把这个自由度停在种子附近，但**不提供任何接触几何引导**。后续系统（AnyTeleop、Bunny-VisionPro）加入姿态正则化来*缓解*模糊性。

MICTF 从一个不同的问题出发：**机器人应当维持什么样的接触，又是怎样的形态学几何在强制它？**

## 核心 insight

> 任何从低维任务目标到冗余运动链的重定向映射，都会留下一个零空间。正则化方法用姿态先验去填充它。**MICTF 用任务本身的接触几何去填充**，让欠约束的自由度服务于接触维持，而非漂向某个任意默认值。

具体而言，MICTF 用一个 **pad 法向对齐残差**来消费拇指的自由 DOF，强制拇指 pad 面朝对侧表面。原本需要被正则化掉的"运动学冗余"反而成了*维持*接触的机制。

这让贡献的定位发生了转变。MICTF 的价值不是更快的求解器、也不是更丰富的隐空间，而是认识到：

1. **"接触后维持"是与"接近"截然不同的工况** —— 在两个 regime 里用同一个自由空间 cost 的方法，操作者一动就会丢失接触。
2. **手自身 pad 的几何 —— 而非姿态先验 —— 才是在该 regime 下解析冗余的正确约束。**
3. **接触点本身也可以是优化变量** —— MICTF 的曲面扩展同时优化关节角与 pad 表面 UV 坐标，将一个硬可行性问题转化为"关节-曲面"乘积空间上的软拟合。

## 系统架构

```
Quest 3 手部关节
        │
        ▼
┌─────────────────────┐
│  Palm-frame 观测    │  计算拇指-手指距离、curl、接触位置分数
└────────┬────────────┘
         ▼
┌─────────────────────┐
│  贝叶斯模板         │  HMM 前向更新；转移代价由机器人自身工作空间中
│  意图滤波           │  的 FK 几何邻近度决定
└────────┬────────────┘
         ▼
┌─────────────────────┐     ┌──────────────────────┐
│  执行状态机         │────▶│  接近模式：          │  朝规范姿态的自由空间跟踪
│                     │     │  保持模式：          │  移动输入下的约束维持
└────────┬────────────┘     └──────────────────────┘
         ▼
┌─────────────────────┐
│  闭链投影求解器     │  有界非线性最小二乘：pad 法向 + 间隙 + 跟踪残差；
│                     │  元数据可用时启用曲面 UV 联合优化
└────────┬────────────┘
         ▼
  有界关节指令 → MuJoCo / 硬件
```

| 层 | 职责 |
|---|---|
| **形态学导入** | 解析 MJCF，推断指链，通过 SVD 自动从网格面片提取 pad 接触坐标系，导出每只手的容差。 |
| **接触模板** | 离散可执行对象（张开 / 精确捏合 / 多点捏合 / 闭合），携带接触角色、pad 坐标系与求解器容差。 |
| **贝叶斯意图滤波** | 模板上的 HMM 前向更新；转移代价来自工作空间几何邻近度，而非手工 penalty。 |
| **闭链投影** | 有界 NLS 求解器维持所选接触；pad 法向残差通过接触几何而非姿态先验解析拇指冗余。 |

## 它不是什么

- **不是抓取或力闭合方法。** 小 pad 间隙 + 坐标系对齐 ≠ wrench 空间闭合或稳定物体操作。
- **暂不支持耦合/欠驱动手。** 肌腱驱动或协同耦合传动需要执行器空间的执行层（未来工作）。
- **不是学习得到的策略。** 没有训练、没有目标域示教。运行时唯一依据是形态学元数据与几何约束。

## 实验结果

在 **5 只手 × 每手 3 个 session**（共 15 组 VR 回放实验）上评估，全部使用真实 Quest 3/WebXR 日志在 MuJoCo 中回放：

| 手型 | MICTF gap mean | 最优 baseline gap | MICTF Gap@10 | MICTF Acq. |
|------|---------------|-------------------|--------------|------------|
| Wuji Right | **8.3 mm** | 40.3 mm（Joint copy） | 0.788 | 0.722 |
| Allegro Right | **15.4 mm** | 51.0 mm（Joint copy） | 0.803 | 0.667 |
| LEAP Right | **19.2 mm** | 44.4 mm（DexPilot） | 0.484 | 0.533 |
| ORCA Right v1 | **8.9 mm** | 23.6 mm（Joint copy） | 0.841 | 0.278 |
| Sharpa Wave | **13.5 mm** | 39.7 mm（Joint copy） | 0.563 | 0.278 |

关键发现：
- **DexPilot 在自己的指标上赢了**（向量误差）：22–48 mm vs MICTF 的 59–107 mm。这是预期内的 —— MICTF 以向量保真度换取 pad 对齐。
- **MICTF 在接触维持上赢了**（gap、hold-gap）：所有 5 只手上间隙低 2–4 倍。
- **所有 baseline 都无法建立稳定接触**（DexPilot/IK 的 Acq. 在全部手型上均为 0.000）。MICTF 是唯一能过渡到稳定保持的方法。
- **关闭闭链求解器**后 gap 回落到 baseline 水平，证实起作用的是投影层本身而非仅仅是模板选择。

trade-off 是显式的、结构性的：DexPilot 获得更好的端点向量匹配；MICTF 获得 pad 级接触，代价是手型姿态与操作者视觉相似度降低。
