# MICTF —— 形态信息驱动的接触模板框架

**在线、无需训练数据的重定向方法，在精确捏合建立*之后*维持 pad 级接触。**

## 这项工作为什么存在

绝大多数灵巧手重定向方法 —— DexPilot 式向量跟踪、关节复制、指尖 IK —— 回答的都是同一个问题：**"指尖该去哪里？"** 它们在自由空间里表现得很好。可一旦两个 pad 已经接触、操作者还在继续移动，这些表征就不再能回答**"接触面之间应该怎样持续贴合"**。结果是一种常见的失败模式：指尖始终停留在向量目标附近，拇指 pad 却绕"拇指-指尖轴"自由旋转，接触在不知不觉中被撑开。

MICTF 从一个不同的问题出发：**机器人应当维持什么样的接触，又是怎样的形态学几何在强制它？**

## 核心 insight

一个只有 4 个自由度、却只被 3D 拇指尖向量约束的机器人拇指，留有一个未被约束的自由度 —— pad 绕"拇指-指尖轴"的旋转。DexPilot 的平滑正则项把这个自由度停在种子状态附近，但它**不提供任何接触几何引导**。后续系统如 AnyTeleop 加入姿态正则化来*缓解*这种模糊性。

MICTF **直接解决**它：用一个 pad 法向对齐残差强制拇指 pad 面朝对侧表面，从而**消费**掉这个自由度。原本是麻烦、需要被正则化掉的"运动学冗余"，反而成了*维持*接触的机制。

这让贡献的定位发生了转变。MICTF 的价值不是更快的求解器，也不是更丰富的隐空间；而是认识到 **"接触后维持"是与"接近"截然不同的一种工况**，并且手自身 pad 的几何 —— 而非某种姿态先验 —— 才是应当被约束的对象。

## MICTF 是什么

MICTF 是一套面向**位控型**灵巧手的、在线、无需训练数据的重定向框架。它在操作者意图和机器人执行之间，插入一层形态学导出的**接触模板**：

| 层 | 职责 |
|---|---|
| **形态学导入** | 解析 MJCF 手部模型，推断指链，从网格面片自动提取 pad 接触坐标系，导出每只手的运行时容差。 |
| **接触模板** | 离散、可执行的对象（张开 / 精确捏合 / 多点捏合 / 闭合），携带接触角色、局部 pad 坐标系与求解器容差。 |
| **贝叶斯意图滤波** | 一个隐马尔可夫前向更新，从 Quest 3/WebXR 观测中选择当前活跃模板，转移代价由机器人自身工作空间中的几何邻近度决定。 |
| **闭链投影** | 一个有界非线性最小二乘求解器，在操作者持续移动时维持所选接触；其 pad 法向残差通过接触几何消解拇指冗余。 |

一个运行时状态机明确区分了**接近**（朝规范姿态的自由空间跟踪）和**锁定保持**（移动输入下的约束维持）两种工况。正是这种区分，使得方法不会在操作者偏离建立接触的姿态那一刻就立刻丢失接触。

## 它不是什么

为了避免误解，需要明确几点：

- **不是抓取或力闭合方法。** 维持小的 pad 间隙和局部坐标系对齐，本身并不构成 wrench 空间闭合或稳定的物体操作。
- **暂不支持耦合/欠驱动手。** 肌腱驱动或协同耦合的传动需要一层执行器空间的执行层，本框架将其留作未来工作。
- **不是学习得到的策略。** 没有训练、没有目标域示教、没有仿真分布。运行时的唯一依据是形态学元数据与几何约束。

## 支持的手型（离线导入）

同一套导入与求解路径覆盖异构的位控型灵巧手：

- **Allegro Right**、**Wuji Right** —— 完整的 VR 回放验证
- **LEAP Hand**、**ORCA Right v1**、**Sharpa Wave** —— 离线导入与求解器检查
- **Shadow**（仅作为肌腱拓扑诊断）、**Robotiq 2F-85**（非对掌诊断）

不同的手部几何从*同一个*导入器得到不同的模板数量与运行时尺度 —— 无需针对每只手硬编码。

## 结果概览

在两只回放手型上，MICTF 将接触后保持阶段的间隙维持在**个位数毫米**级别，而 DexPilot 式向量重定向、关节复制、指尖 IK 都漂移到**数十毫米**，或根本无法建立紧密接触。关闭闭链投影会使性能回落到基线水平，证明起作用的是投影层本身 —— 而不是规范姿态的查表。

详细的数字、消融以及每只手的求解器诊断将随论文一同公布。
